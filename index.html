Basic Concepts

Addressing · MAR & MDR · Access Time · Cycle Time

Memory Size & Addressing

The maximum memory size is set by the number of address bits the CPU can generate.

Rule k address bits → 2k addressable locations
Example: A 16-bit computer generates 16-bit addresses → can address up to 216 = 64K locations. A 32-bit address space → 232 = 4G locations.

Byte-Addressable: Most modern computers address individual bytes. Endianness determines byte ordering — little-endian (Intel), big-endian (68000), bi-endian (ARM).

MAR — Memory Address Register

Holds the address of the memory location to be read from or written to.

If MAR is k bits wide → memory can have up to 2k locations

MDR — Memory Data Register

Holds the data being transferred to or from memory.

If MDR is n bits wide → n bits are transferred per memory cycle (= word length)

Memory Read & Write Operations

🟢 READ Operation:

  1. Load address of required location into MAR
  2. Set R/W line to 1 (Read)
  3. Memory places data on data lines
  4. Memory asserts MFC (Memory Function Completed) signal
  5. Processor loads data from data lines into MDR

🔴 WRITE Operation:

  1. Load address into MAR
  2. Load data into MDR
  3. Set R/W line to 0 (Write)
  4. Wait for MFC signal

Memory Access Time ⏱

Time elapsed between initiation and completion of a memory operation.

Example: time between sending a READ command and receiving MFC signal

Memory Cycle Time 🔄

Minimum delay between initiations of two successive memory operations.

Cycle Time > Access Time (always slightly longer)

RAM — Random Access Memory

A memory unit where any location can be read or written in a fixed amount of time, regardless of address. This distinguishes it from serial access devices like magnetic tapes and disks (which depend on physical position).

RAM Types Overview

Memory Organization · Chip Structure

Memory Chip Organization — 16×8 Example

A 16×8 chip stores 16 words × 8 bits = 128 bits. Cells are arranged in an array.

  • Each row = one memory word; all row cells share a word line
  • Word line is connected to the address decoder
  • Each column has a Sense/Write circuit connected to a bidirectional data line
  • R/W line specifies the operation; CS (Chip Select) selects the chip
  • 16×8 chip → 16 pins: 4 address + 8 data + 2 control + 2 power

1K × 1 Memory Organization

1K cells = 1024 bits. Can be organized as 128×8 (19 pins) or 1K×1 (15 pins).

In 1K×1: 10-bit address split into two 5-bit groups → row address (selects 32 cells) + column address (picks 1 of the 32). Reduces pin count.

SRAM Cell — How it works

Built from two cross-connected transistor inverters forming a flip-flop. Connected to word line (W) and two bit lines (b, b') by transistors T1 and T2.

Read: Activate word line → T1 & T2 close → sense b and b' → state 1: b=high, b'=low
Write: Set b and b' to desired values → activate word line → forces cell into that state

When word line = ground, transistors OFF → latch retains state indefinitely.

DRAM Cell — How it works

A single transistor + capacitor. The capacitor stores charge = bit value. Charge leaks over time → must be refreshed.

Write: Drive bit line → select row → capacitor charges/discharges
Read: Precharge bit line → select row → cell shares charge → sense tiny voltage change → write value back (destructive read!)
Refresh: Just do a "dummy read" on every row periodically
🔁

DRAM Variants

Async → Sync → Fast Page Mode → DDR

Asynchronous DRAM

The original DRAM type. Addresses are sent in two parts using multiplexed address pins to reduce pin count:

  • RAS (Row Address Strobe) — latches the row address (high-order bits)
  • CAS (Column Address Strobe) — latches the column address (low-order bits)
Timing is NOT synchronized with processor clock — controller generates RAS/CAS independently.

Fast Page Mode (FPM)

When the DRAM row is selected, all cells in that row are sensed and latched. To access consecutive bytes in the same row:

No need to re-select the row! Apply different column addresses using CAS repeatedly → much faster for sequential/block access. This is called "fast page mode".

Synchronous DRAM (SDRAM)

Operation is directly synchronized with the processor clock. Uses address/data registers (latches) at inputs/outputs. Key features:

  • Burst mode: Column counter auto-increments → data placed on lines every clock cycle
  • No need for external CAS pulses for successive columns
  • All actions triggered on rising edge of clock
  • Mode register configures burst length and operation
  • Built-in refresh circuitry (each row refreshed every 64 ns)
Latency (example) RAS assertion → first data word = 5 clock cycles → Latency = 5

DDR SDRAM — Double Data Rate

Same cell array as SDRAM, but transfers data on both rising and falling edges of the clock.

Result: Same latency, but bandwidth is effectively doubled for long burst transfers.

Cell array organized in 2 or 4 banks. Consecutive words stored in different banks (interleaving) → two words accessed simultaneously per clock cycle.

Latency vs Bandwidth

⏱ Latency

Time to transfer the first word of data. In block transfers, this is the startup cost.

📶 Bandwidth

Total bits/bytes transferred per second. Depends on: memory speed, bus speed, and number of bits transferred in parallel.

ROM Family

Non-Volatile Memory Types
Why ROM? SRAM and DRAM are volatile — they lose data when power is off. ROMs are non-volatile — they retain data permanently. Used for BIOS, firmware, boot programs.

ROM — Read-Only Memory PERMANENT

Data is written at manufacturing time. Cannot be changed. Reading works the same as RAM.

Cell: Transistor connected (or not) to ground at point P → stores 0 or 1. To read: activate word line; voltage drops = 0, stays = 1.

PROM — Programmable ROM WRITE ONCE

User can write data once using high-current pulses. A fuse at point P is blown to program.

  • Initially all 0s; user inserts 1s by blowing fuses
  • Irreversible — once written, cannot be changed
  • Needs special equipment to program

EPROM — Erasable Programmable ROM UV ERASABLE

Can be erased and reprogrammed. Uses a special transistor that can be disabled by trapping charges.

  • Erase by exposing chip to UV (ultraviolet) light — dissipates trapped charges
  • Useful during development phase when frequent changes are needed
Disadvantages: Chip must be physically removed from circuit for reprogramming. UV erases the entire chip, not individual cells.

EEPROM — Electrically Erasable PROM ELECTRICALLY ERASABLE

Can be programmed and erased electrically — no removal from circuit, no UV needed.

  • Can erase individual cells selectively (unlike EPROM)
Disadvantages: Different voltages needed for erase/write/read. Writing is much slower than reading.

Flash Memory BLOCK ERASE

Similar to EEPROM but reads single cells, writes/erases entire blocks. Key properties:

  • High density, low power, high capacity
  • Block-level erase — entire block erased at once (not per-cell)
  • Used in USB drives, SSDs, phones, cameras, MP3 players
  • Single chips insufficient → packaged as Flash Cards (plug-in) or Flash Drives (SSD-like)
Flash drives: shorter seek times, lower power, insensitive to vibration compared to hard disks.

Cache Memory

Locality · Hit/Miss · Write Policies · Coherence

Why Cache?

The processor is much faster than main memory. Without cache, the processor spends most of its time waiting for data. Cache is a small, fast memory placed between CPU and main memory to hide this speed gap.

Cache hit rate > 90% is achievable with a good cache design — meaning the processor finds data in cache 9 out of 10 times.

Locality of Reference 📍

Cache works because programs exhibit locality — they reuse the same data and code repeatedly.

Temporal Locality: A recently accessed location will likely be accessed again soon. (e.g., loop variables, frequently called functions)
Spatial Locality: Locations near a recently accessed location are likely to be accessed soon. (e.g., array elements, sequential instructions)
Sequential Locality: Instructions tend to be accessed sequentially (one after another).

How Cache Works — Overview

  1. CPU requests a memory location
  2. Check cache for this data (check tags)
  3. Cache HIT: Data found → serve directly from cache (fast!)
  4. Cache MISS: Data not found → fetch entire block from main memory into cache
  5. Deliver requested word to CPU
Cache block / Cache line: A contiguous group of memory words transferred together. Exploits spatial locality.

Cache HIT ✅

Read Hit: Data found in cache → processor gets it directly. Main memory NOT involved.

Write Hit — two protocols:

Write-Through: Update cache AND main memory simultaneously. Simple, reliable. Good when writes are infrequent.
Write-Back (Copy-Back): Update only the cache. Set the dirty bit. Write to main memory only when the block is replaced.

Cache MISS ❌

Read Miss: Fetch block from memory → load into cache → send data to CPU.

Load-Through / Early-Restart: Forward the requested word to CPU as soon as it arrives, without waiting for the entire block.

Write Miss:

Write-Through: Update main memory directly.
Write-Back: Bring block into cache first, then update the word.

Cache Coherence Problem 🔄

When multiple copies of data exist (cache + main memory + disk), they can become inconsistent.

Valid Bit: Each cache block has a valid bit. If 1 → block is valid. If 0 → block is stale/invalid. On power-on, all valid bits = 0.
Case 1 (DMA modifies disk → main memory): Cache copy becomes stale. Solution: Set valid bit = 0 to mark cache block as stale.
Case 2 (Cache modified, DMA reads main memory to disk): Main memory has old data. Dirty bit is set. Solution: Force write-back before DMA transfer.

Replacement Algorithms

Which cache block to evict when cache is full?
When: Cache is full and a new block needs to be loaded. Only applies to Associative and Set-Associative caches (Direct mapping has only one choice).

LRU — Least Recently Used BEST PERFORMANCE

Replace the block that was accessed least recently. Based on temporal locality — recently used data is likely needed again.

Implemented using a USE bit (for 2-way): set USE=1 when accessed, clear others. For more ways: need timestamps → expensive.

FIFO — First In, First Out

Replace the block that has been in the cache the longest, regardless of usage frequency.

Easy to implement as a circular buffer. May evict frequently used blocks just because they arrived first.

LFU — Least Frequently Used

Replace the block with the fewest accesses. Needs a hit counter per block.

Random Replacement

Replace a randomly chosen block. Surprisingly good in practice, simple to implement.

Performance & Interleaving

Hit Rate · Miss Penalty · Memory Interleaving

Key Performance Definitions

TermDefinition
HitData found at the given memory level
MissData NOT found → must go to the next level
Hit Rate (h)% of accesses that result in a hit
Miss Rate1 − Hit Rate
Hit TimeTime to access data at a given level (on a hit)
Miss PenaltyExtra time to fetch data from next level on a miss
Average Access Time (2-level cache) Tavg = h₁·c₁ + (1−h₁)·h₂·c₂ + (1−h₁)·(1−h₂)·M

Where h₁ = L1 hit rate, c₁ = L1 access time, h₂ = L2 hit rate, c₂ = L2 access time, M = main memory time.

Improving Performance

Improve Hit Rate: Increase block size (exploits spatial locality). But too large → fewer cache lines, more conflicts.
Reduce Miss Penalty: Use load-through / early-restart — send the requested word to CPU as soon as it arrives, don't wait for full block.

Memory Interleaving 🔀

Divide memory into multiple independent memory modules, each with its own ABR (Address Buffer Register) and DBR (Data Buffer Register). Arrange addressing so consecutive addresses go to different modules.

Key benefit: Consecutive memory accesses go to different modules → can be served in parallel → increases effective memory bandwidth.
Low-order interleaving: Low-order address bits select the module. Consecutive words → consecutive modules → parallel access possible. Best for block transfers.
High-order interleaving: High-order bits select the module. Consecutive words → same module. Only one module active during block transfer.
Miss Penalty Example (1-word bus) Miss penalty = 1 (addr) + 4×15 (DRAM) + 4×1 (transfer) = 65 cycles
With 4-bank interleaving Miss penalty = 1 + 1×15 + 4×1 = 20 cycles (much better!)

Memory Controller

The bridge between processor and DRAM

Why a Memory Controller?

DRAM uses multiplexed addresses (row address first, then column address, through the same pins). But the processor sends all address bits at the same time. The memory controller bridges this gap.

Memory Controller Functions

  • Sits between processor and DRAM memory
  • Accepts complete address + R/W from processor
  • Splits address → sends row address first with RAS signal
  • Then sends column address with CAS signal
  • Sends R/W and CS signals to memory
  • Data lines are connected directly (no buffering)
  • Contains a refresh counter to periodically refresh all DRAM rows within the specified time limit

Refresh Overhead Calculation

Example: SDRAM with 8192 rows, 4 clock cycles per row, 133 MHz clock, 64 ms refresh period.
Total refresh cycles 8192 × 4 = 32,768 cycles
Time for refresh 32,768 / 133×10⁶ = 246×10⁻⁶ sec = 0.246 ms
Refresh overhead 0.246 / 64 = 0.0038 = 0.38% of total time
Only 0.38% overhead — negligible! DRAM refresh has very little impact on performance.

Quick Quiz

Test yourself — click "Show Answer" for each question
Q1. A 20-bit address bus can access how many memory locations?
220 = 1M (1,048,576) locations. Rule: k address bits → 2k locations.
Q2. What is the difference between Memory Access Time and Memory Cycle Time?
Access Time: Time from initiation to completion of one operation (READ → MFC).
Cycle Time: Minimum delay between starting two successive operations. Cycle time is always slightly longer than access time.
Q3. Why does DRAM need to be refreshed, but SRAM does not?
DRAM stores bits as charge on a capacitor, which leaks over time. If not refreshed (~every 64ms), data is lost.
SRAM uses flip-flops (cross-coupled inverters) which actively maintain state — no charge leaks, no refresh needed.
Q4. In direct mapping, memory block 200 maps to which cache line? (128 cache lines total)
B = 200 mod 128 = 72
Block 200 maps to cache line 72.
Tag = 200 / 128 = 1 (this is the 2nd group of 128 blocks).
Q5. What is "thrashing" in the context of cache memory?
Thrashing occurs in direct-mapped caches when two memory blocks that both map to the same cache line are accessed alternately. Each access evicts the other, causing constant cache misses even though the cache has free space for other blocks.
Q6. Which ROM type requires physical removal from the circuit for reprogramming?
EPROM (Erasable Programmable ROM). It is erased by exposing the chip to UV (ultraviolet) light, which requires removing it from the circuit. EEPROM and Flash can be erased electrically without removal.
Q7. Write-back vs Write-through — which is better for performance and why?
Write-back is generally faster because writes only go to cache, not main memory on every write. Main memory is updated only when the block is evicted (indicated by the dirty bit). This reduces memory bus traffic.

Write-through is simpler and always keeps memory consistent, but generates more memory traffic.
Q8. In set-associative mapping with 4-way sets and 32 sets total, where does memory block 50 go?
Set number = 50 mod 32 = 18
Block 50 maps to set 18. Within set 18, it can go into any of the 4 lines (4-way). The replacement algorithm decides which line if all 4 are full.
Q9. What is the "valid bit" and when is it set to 0?
The valid bit is a flag for each cache block. It is set to 1 when a block is loaded from main memory (meaning the block contains valid data).

It is set to 0:
  • When power is first turned on
  • When a DMA transfer modifies main memory, making the cached copy stale (cache coherence problem)
Q10. Why does DDR SDRAM have double the bandwidth of standard SDRAM?
DDR SDRAM transfers data on both the rising and falling edges of the clock signal, instead of only the rising edge. This doubles the number of transfers per second with the same clock frequency → double the bandwidth. Latency remains the same.
